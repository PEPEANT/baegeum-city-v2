# 배금도시 v2 월드 시간 설계

결론: 배금도시의 시간은 낮밤 효과가 아니라 `가상 인터넷`, `NPC 스케줄`, `도박장 피크타임`, `뉴스/주식/커뮤니티 갱신`, `온라인 라운드`의 기준축이다.

## 현재 구현

- `src/systems/world-clock.js`에 `WorldClock`을 추가했다.
- 기본 시작 시간은 `DAY 01 08:00`이다.
- 현재 공식 기본 속도는 실제 1초 = 게임 내 1분이다.
- 이 느린 기본 속도는 노가다 쿨타임, 배고픔/에너지 감소, 주식 tick, 랭킹 집계가 너무 빠르게 튀지 않게 하기 위한 기준이다.
- 빠른 테스트가 필요하면 `new WorldClock({ minutesPerSecond: 60 })`처럼 명시적으로 override한다.
- 중앙 상단 시간 HUD는 `WorldClock`을 표시한다.
- MammonCity 휴대폰 셸의 고정 `08:00 / TURN 01` 표시를 `WorldClock` 기반으로 바꿨다.
- 도시 화면에는 시간대별 최소 조명 오버레이를 그린다.

## 시간대

- `새벽`: 05:00-06:59
- `낮`: 07:00-17:59
- `저녁`: 18:00-20:59
- `밤`: 21:00-04:59

시간대는 지금은 조명과 UI 표시만 바꾸지만, 이후 NPC/인터넷/사건 조건의 기준이 된다.

## 반복 이벤트

1차 반복 이벤트는 아래처럼 잡는다.

```js
[
  { id: "morning-news", minuteOfDay: 480, target: "internet.news" },
  { id: "lunch-crowd", minuteOfDay: 720, target: "npc.schedule" },
  { id: "casino-peak", minuteOfDay: 1140, target: "venues" },
  { id: "night-rumor", minuteOfDay: 1380, target: "internet.community" }
]
```

규칙:

- 이벤트는 클라이언트에서 먼저 미리보기로 동작할 수 있다.
- 온라인에서는 서버 시간이 이벤트 발생 기준이 된다.
- 인터넷 글/뉴스/주식/루머는 생성 시각을 반드시 가진다.
- 기본 속도는 `tools/smoke-world-clock.cjs`에서 검증한다. 문서와 코드 속도가 다시 갈라지면 이 검증을 먼저 고친다.

## 가상 인터넷 연결

가상 인터넷은 앱 목록이 아니라 시간에 따라 갱신되는 정보망이다.

- 뉴스: 특정 시간마다 경제/사건 뉴스가 올라온다.
- 커뮤니티: 밤 루머, 카지노 후기, 도박장 소문이 시간대에 따라 올라온다.
- 주식: 뉴스/루머/시간 이벤트를 가격 변동 이유로 사용한다.
- 은행/칩: 영업시간, 점검 시간, 거래 기록 시간이 필요하다.

따라서 폰 앱 구현 전 `WorldClock`이 먼저 있어야 한다.

## NPC 스케줄 연결

NPC는 현재 랜덤 배회만 한다. 이후에는 시간대별 목적지를 가진다.

```text
08:00 출근/이동
12:00 식사/상점 밀집
19:00 카지노 거리 증가
23:00 위험 NPC/루머 NPC 등장
```

NPC 스케줄은 `WorldClock`을 읽되, 첫 구현은 클라이언트 미리보기로 둔다.

## 온라인 규칙

온라인 상태에서는 서버 시간이 권위다.

- 라운드 시작/마감
- 채팅 rate limit
- 랭킹 집계
- 재접속 유예
- 경마/이벤트 결과

클라이언트 시간은 UI 표시와 예측용일 뿐이다. 실제 돈/결과/랭킹과 연결되는 시간 판정은 서버가 확정한다.
